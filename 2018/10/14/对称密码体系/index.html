对称密码体系

主要特征：加密密钥和解密密钥相同
主要研究课题：密钥的产生/管理

流密码

流密码的安全强度完全取决于密钥的随机性和不可预测性
核心问题是：密钥流的产生
保持收发两端密钥流的精确同步是关键问题

基本思想：利用密钥k和密钥流发生器f生成一个密钥流
密钥流：Zi = f(k,δi)，δi是加密器中的记忆元件在时刻i的状态，f是以密钥k和δi作为输入参数的函数；

同步流密码
加密器中记忆元件的存储状态δi独立于明文字符
加密器：1、密钥流产生器 2、加密变换器

密钥流生成器的设计原则：

  • 足够长的周期
  • 高线性复杂度
  • 统计性能良好
  • 足够的“混乱”
  • 足够的“扩散”
  • 抵抗不同形式的攻击

在对称密码中，最重要的部分就是非线性部件

LFSR：线性反馈移位寄存器——流密码中产生密钥流的重要部分
级数stages：存储单元数n
状态state：n个存储单元的存数
反馈函数：f(state)
线性反馈移位寄存器：f是线性函数
非线性反馈移位寄存器：f是非线性函数

非线性序列
LFSR的输出不能直接作为函数，但是可以驱动一个非线性组合函数来产生密钥流：LFSR用了保证密钥的周期长度，平衡性等因素/非线性组合函数用来保证密钥流的各种密码性质，以抵抗各种可能的攻击

J-K触发器：是一个非线性的器件，输出不仅依赖于输入还依赖于前一个输出位

序列密码算法RC4

RC4的工作模式（OFB模式）

输出反馈模式（Output feedback, OFB）可以将块密码变成同步的流密码。它产生密钥流的块，然后将其与明文块进行异或，得到密文。与其它流密码一样，密文中一个位的翻转会使明文中同样位置的位也产生翻转。这种特性使得许多错误校正码，例如奇偶校验位，即使在加密前计算，而在加密后进行校验也可以得出正确结果。

随机密钥
1、密钥调度算法KeyScheduleAlgorithm
2、伪随机数生成算法PRGA
基本流程
1、密钥调度算法将一个随机密钥变换成一个初始置换
初始置换：

* 初始化S盒，线性填充S0=0, S1=1, …, S255=255
* S盒置换，交换Si和Sj，不断重复密钥K(L字节长) 
   j = (j + Si + K(i mod L)位)mod256.   j初始值为0

2、伪随机数生成算法利用初始置换生成一个伪随机数处序列

* 伪随机序列的生成：
* 伪代码：
    I := 0
    j := 0
    while GeneratingOutput:
    I := (I + 1) mod 256
    j := (j + S[I]) mod 256
    swap values of S[I] and S[j]
    K := S[(S[I] + S[j]) mod 256]
    output K
    endwhile

3、用伪随机数处理序列于plaintext进行XOR操作——加密/解密

DES（Data Encryption Standard）

DES是一种将64bit明文加密为64bit密文的对称密码算法，它的密钥长度是56位（每隔7bit设置一个校验位）DES是以64bit明文为一个单位进行加密的，这64bit单位称为 分组，以分组为单位的密码算法称为 分组密码(block cipher)

DES：加解密流程

  1. 输入64bit明文数据
  2. 初始置换IP
  3. Feistel网络，在密钥控制下16轮迭代
  4. 交换左右32bit
  5. 初始逆置换IP^(-1)
  6. 输出64bit密文数据

此图来自Pro杨秋伟(HNU)在2018年秋季学期密码学的课程PPT

DES:初始置换

初始置换IP和初始逆置换IP^(-1)在密码意义上作用不大，主要意义在于打乱原来输入的ASCII码字的划分关系

IP:将原输出的不同位置映射到不同的位置上

DES:乘积变换

乘积变换是DES算法的核心部分

  • 将经过IP置换后的数据分成32bit的左右两组数据

分组：
按照IP置换后的序列以8 * 8的方式排列
将左边4列作为左边，将右边4列作为右边

  • 每次迭代只对右边的32bit进行一系列的加密变换，在此轮迭代即将结束的时候，将左边的32bit与右边的32bit逐位进行XOR运算，作为下一轮迭代时右边的段

    • R(32bit)

    • 扩展运算E(48bit)

      通过选择扩展运算将输入的32bit扩展为48bit,通过一张扩展表实现

    • 与此轮的轮密钥进行XOR操作

    • 压缩运算S(32bit)

      将前面送来的48bit数据自左到右分成8组，每组分为6bit,然后并行送入8个S盒，每个S盒，有6bit输入，4bit输出
      e.g. 输入：1 0 1 1 0 0
      将输入的中间四位：0 1 1 0作为该S盒的横坐标，即6
      将两边的两位 1 0 作为该S盒的纵坐标，即2就可以得到一个十进制数字，将其用4位二进制数表示即可得到输出的4bit

    • 置换运算P(32bit)

      将S1到S8输出的32bit进行坐标置换

    • 与左边L(32bit)进行XOR操作

    • 得到下一轮的R(32bit)
    • 将此轮最开始的R(32bit)直接作为下一个的L(32bit)

  • 将原来右边未经变换的段直接送到左边的寄存器作为下一轮迭代时左边的段

  • 在每一轮迭代时，右边的段要经过选择扩展运算E，密钥加密运算，选择压缩运算S，置换运算P和左右混合运算

子密钥产生器

在初始密钥中，有8bit为校验位

  • 置换选择PC：将56bit密钥送入置换选择PC1进行坐标置换
  • 循环移位置换：将上述结果分为两组28bit，分别送入C，D寄存器，在每次的迭代中，C和D寄存器将存数分别进行左循环移位
  • 置换选择PC2：每次移位之后，将C和D寄存器中的原存数送给置换选择PC2，将C中第9，18，22，25位和D中第7，9，15，26位删除，并将其余数字置换位置后送给48bit作为该轮的子密钥

DES的结构(Feistel网络)

在Feistel网络中，加密的各个步骤称为 轮，

一轮的具体计算步骤

  1. 将输入的数据分为左右两部分
  2. 将输入的右侧直接发送给输出的右侧
  3. 将输入的右侧发送给轮函数
  4. 轮函数将右侧数据和子密钥计算后得到一串bit序列
  5. 将得到的bit序列与左侧数据进行XOR操作，得到结果为输出的左侧

在两轮之间将左侧和右侧进行交换

三重DES： 执行三次DES加密，根据使用的密码不同有更加详细的分类

AES(Rijndael)算法也是由多个轮组成的其中每一轮分为：

  1. SubBytes
  2. ShiftRows
  3. MixColumns
  4. AddRoundKey

SubBytes

Subytes就是通过一张256个值的表，以每个字节(8bit)为索引值，找到S-Box中的替换值。

*对称密码中提高非线性性的手段

ShiftRows

ShiftRows就是将行按照一定的规则向左平移，并且每一行平移的字节数是不同的。

MixColumns

MixColumns是将一个4byte的值进行bit运算，将其变为另一个4字节的值。

AddRoundKey

AddRoundKey就是将MixColumns的输出值与轮密钥进行XOR

AES算法简介

  • AES是分组密码，但是AES并不具有Feistel结构，具有Square结构
  • 加密与解密相似但是不对称
  • 密钥长度和分组长度可变，可以被分别的指定为：128bit,192bit,256bit
  • 基于较好的数学理论，例如：伽罗瓦域
  • 结构简单，速度较快
  • 能在多种平台上以较快的速度实现
